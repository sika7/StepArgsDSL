# 📘 StepArgs DSL 言語仕様書（v0.2）

**StepArgs DSL（StepArgs Domain-Specific Language）** は、人間にも機械にも読みやすく、簡潔な形式でステップベースの処理を記述するための軽量DSLです。
主に LLM や自立型エージェントにおけるタスク指示・中間構造として使用されます。

## ✅ 基本構文

### 🔹 文字エンコーディング

- **エンコーディング**: UTF-8（BOM無し）
- **改行コード**: LF（`\n`）推奨、CRLF（`\r\n`）も対応
- **特殊文字**: Unicode全般に対応（絵文字、記号、多言語文字含む）

### 🔹 命名規則と制約

#### ステップ名

- **使用可能文字**: Unicode文字（空白文字、制御文字以外）
- **長さ制限**: 1～128文字
- **禁止文字**: `---`、改行文字、タブ文字
- **例**: `記事検索`、`ArticleSearch`、`データ分析_v2`、`🤖AI処理`

#### 引数名

- **使用可能文字**: Unicode文字（`:`, `[`, `]`, `<`, `>`以外）
- **長さ制限**: 1～64文字
- **禁止文字**: コロン`:`, 角括弧`[]`, 不等号`<>`
- **例**: `キーワード`、`maxLength`、`出力形式`、`🎯target`

#### 引数値

- **使用可能文字**: 制限なし（改行も含む）
- **長さ制限**: 単行：16,384文字、複数行：1,048,576文字
- **エスケープ**: 不要（そのまま格納）

### 🔹 多言語サポート

#### 対応言語

- **完全対応**: 日本語、英語、中国語、韓国語、アラビア語
- **基本対応**: Unicode文字を使用する全言語
- **混在**: 同一ファイル内で複数言語の混在可能

#### 言語固有の考慮事項

- **RTL言語**: アラビア語等も文字列として正常に処理
- **結合文字**: 絵文字の肌色修飾子等も1文字として扱う
- **正規化**: NFC正規化推奨（必須ではない）

### 🔹 ステップブロックの記述

```
--- ステップ名 ---
```

- 各処理単位（タスク/関数など）を `---` で囲みます。
- ステップ内に複数の引数行を記述します。

### 🔹 引数行の記述

#### 単行引数

```
ステップ名[引数名:値]
```

#### 複数行引数（ヒアドキュメント形式）

```
ステップ名[引数名:<<<
複数行の内容
改行も含められます
>>>]
```

- `ステップ名`：ブロックと一致している必要があります。
- `引数名`：1語または短いフレーズ（日本語/英語どちらも可）
- `値`：任意の文字列、URL、数値、記号など（複数行対応）

## 🧾 サンプル

### 単行引数の例

```
--- 記事検索 ---
記事検索[キーワード:AI技術]
記事検索[期間:過去3日間]

--- 記事取得 ---
記事取得[URL:https://example.com/ai-news]

--- 要約 ---
要約[文字数上限:300]
```

### 複数行引数の例

```text
--- 文書処理 ---
文書処理[対象:<<<
人工知能（AI）は、近年急速に発展している技術分野です。
機械学習、深層学習、自然言語処理など、様々な分野で
革新的な進歩が見られています。

特に大規模言語モデル（LLM）の登場により、
人間に近い文章生成や理解が可能になりました。

>>>]
文書処理[出力形式:markdown]

--- API設定 ---
API設定[エンドポイント:https://api.example.com/v1/chat]
API設定[リクエストボディ:<<<
{
  "model": "claude-4-sonnet",
  "messages": [
    {
      "role": "user",
      "content": "Hello, world!"
    }
  ],
  "max_tokens": 1000
}
>>>]
```

## 🔧 特徴と利点

| 特徴                  | 説明                                 |
| --------------------- | ------------------------------------ |
| ✅ 人間に優しい構文   | JSON より書きやすく視認性が高い      |
| ✅ 正規表現で解析可能 | 小規模でも LLM パーサーを自作できる  |
| ✅ モジュール分離可能 | ステップ単位に処理分岐しやすい       |
| ✅ LLM に強い         | JSON 出力崩壊の回避に有効            |
| ✅ 複数行対応         | 長文・JSON・構造化データも自然に記述 |

## 🧰 使用例

### 基本的なタスク指示

```
--- 翻訳 ---
翻訳[入力言語:日本語]
翻訳[出力言語:英語]
翻訳[内容:こんにちは、世界]

--- 要約 ---
要約[最大文字数:100]
```

### 複雑なデータ処理

```text
--- データ分析 ---
データ分析[設定:<<<
{
  "source": "analytics.csv",
  "filters": {
    "date_range": "2024-01-01,2024-12-31",
    "categories": ["tech", "business"]
  },
  "output": {
    "format": "chart",
    "type": "line_graph"
  }
}
>>>]
データ分析[グラフタイトル:2024年度売上推移]
```

### LLMエージェント連携

```
--- Web検索 ---
Web検索[クエリ:最新のAI技術動向]
Web検索[サイト制限:scholar.google.com]
Web検索[結果件数:10]

--- 記事分析 ---
記事分析[分析観点:技術革新性]
記事分析[重要度判定:高・中・低]

--- レポート生成 ---
レポート生成[形式:Markdown]
レポート生成[文字数:1500]
レポート生成[構成:<<<
# AI技術動向レポート
## 1. 注目技術
## 2. 市場への影響
## 3. 今後の展望
>>>]
```

## 💡 設計思想

### 人間中心設計

- **直感的な構文**: プログラミング経験がなくても理解しやすい
- **視認性の高さ**: ステップ区切りが明確で、長い処理も把握しやすい
- **自然言語親和**: 日本語・英語混在でも違和感のない記述

### LLM最適化

- **出力安定性**: JSONよりもフォーマット崩壊が起きにくい
- **段階的処理**: ステップ単位での処理制御が容易
- **コンテキスト効率**: 構造が明確で、LLMが理解しやすい

### 拡張性

- **モジュール性**: ステップ単位での追加・削除が容易
- **多様なデータ**: 文字列、JSON、コード、長文など幅広く対応
- **言語非依存**: プログラミング言語に依存しない中間表現

## 🎯 適用領域

### LLMエージェント

- タスク分解と実行指示
- 多段階推論の構造化
- エージェント間の情報交換

### ワークフロー記述

- 業務プロセスの可視化
- 自動化シナリオの定義
- 条件分岐を含む複雑な処理

### プロンプトエンジニアリング

- 構造化プロンプトの作成
- Few-shot学習の例示
- チェーン・オブ・ソート（CoT）の実装

### データ処理パイプライン

- ETL処理の定義
- データ変換ルールの記述
- 複数ソースからの集約処理

## まとめ

StepArgs DSL は、構造化されたタスク記述を軽量・自然に行うためのDSLです。JSON より簡潔で、LLM に安定した出力を促します。単行・複数行引数の両方に対応し、幅広い用途で活用可能です。構文は正規表現で簡単に処理可能で、AIエージェント、関数実行、ワークフロー記述、プロンプト設計などに活用できます。
